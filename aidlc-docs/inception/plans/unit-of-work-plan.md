# Unit of Work Plan

## 분해 전략
NestJS monolith 내 모듈화 구조. 기능 도메인별로 unit을 분리하되, 의존성 순서에 따라 순차 개발.
TDD 적용을 위해 각 unit은 독립적으로 테스트 가능해야 함.

- [x] 1. 기반 unit 정의 (DB, 인증, 공통)
- [x] 2. 핵심 비즈니스 unit 정의 (상품, 주문, 재고)
- [x] 3. 신규 기능 unit 정의 (리뷰, 인플루언서, 결제, 실시간)
- [x] 4. 확장 기능 unit 정의 (멀티채널, 바코드, AI 예측)
- [x] 5. 프론트엔드 unit 정의
- [x] 6. 인프라 unit 정의
- [x] 7. Unit 간 의존성 매트릭스 생성
- [x] 8. Story-Unit 매핑 생성
